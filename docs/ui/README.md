# UI Docs

Canonical UI doctrine for the visual-first FlavorOS 1.5 MVP.

UI docs follow `docs/planning/current_build_plan.md`. If a UI doc conflicts with the current build plan, update the UI doc.

Current UI build priorities:

1. visualization and surfaces,
2. database-backed display state,
3. integration/provider posture,
4. onboarding surfaces,
5. provider ingestion visibility,
6. agent workflow surfaces.
